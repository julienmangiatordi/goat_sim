# GOAT Frame Design Files Explanation

## Files Overview

### 1. `generate_rings.py`
**Purpose**: Generates the individual components of the frame as separate OBJ mesh files.
- Creates `ring_A.obj`: A torus ring in the X-Z plane (tilted at +15°).
- Creates `ring_B.obj`: A torus ring in the Y-Z plane (tilted at -15°).
- Creates `strut_top.obj` and `strut_bot.obj`: Vertical connecting struts at the ends.

**Why this approach**: Allows for modular assembly in simulations, where each part can be simulated separately or together. Uses trimesh library for accurate torus geometry. The tilting prevents perfect alignment, making the structure more realistic.

### 2. `frame_drop_fem.py`
**Purpose**: Finite Element Method (FEM) simulation of the frame components dropping onto a rigid ground plane.
- Loads the OBJ files generated by `generate_rings.py`.
- Uses Genesis FEM solver with elastic material properties for fiberglass.
- Simulates gravity and deformation upon impact.

**Why FEM**: FEM is suitable for simulating deformable solids like the thin fiberglass tubes. It captures bending, stretching, and contact deformations accurately. The implicit solver ensures numerical stability for the high Young's modulus (5e6 Pa, reduced for stability).

### 3. `generate_lens_frame_genesis.py`
**Purpose**: Standalone mesh generator for the complete lens frame OBJ file (`goat_lens_genesis.obj`).
- Uses only NumPy (no external dependencies like trimesh).
- Creates two circular rings in perpendicular planes connected by cylindrical struts.
- Exports as OBJ for use in Genesis or other simulators.

**Why this way**: Provides a dependency-free way to generate the mesh. The rings are discretized into cylindrical segments, and struts connect them at regular angles. This creates the characteristic "lens" shape where the rings cross at four points.

### 4. `frame_scratch.py`
**Purpose**: Comprehensive simulation script that generates the frame mesh on-the-fly and simulates deformation in a constrained environment.
- Generates the mesh using trimesh (similar to `generate_lens_frame_genesis.py` but with trimesh).
- Adds rigid obstacles (walls) to simulate a gap-crossing scenario.
- Uses FEM with implicit solver and visualizes deformation.

**Why this approach**: Integrates mesh generation and simulation for testing the frame's ability to deform and pass through narrow spaces. The obstacles represent real-world terrain challenges. The script includes detailed comments explaining the GOAT paper's Figure 2.B design.

## Design Rationale

- **Lens Shape**: Two perpendicular rings provide structural integrity in multiple directions while remaining lightweight.
- **Fiberglass Material**: High strength-to-weight ratio, modeled with reduced Young's modulus for numerical stability in simulations.
- **FEM Simulation**: Chosen over MPM (Material Point Method) for better handling of thin structures and contact.
- **Modular vs. Monolithic**: Separate parts allow testing assembly and individual component behavior, while full mesh enables system-level deformation analysis.

## Generated Files
- `ring_A.obj`, `ring_B.obj`, `strut_top.obj`, `strut_bot.obj`: Individual components.
- `goat_lens_genesis.obj`: Full frame mesh.
- `goat_lens_simulation.obj`: Full frame with simulation parameters.
- `mesh_gen.log`: Log from mesh generation runs.
